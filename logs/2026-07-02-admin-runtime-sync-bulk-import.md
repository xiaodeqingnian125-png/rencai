# 2026-07-02 管理员运行时同步与批量导入导出

## 改动范围

- 新增 `miniprogram/data/mock-store.js`，基于静态表克隆运行时 mock 数据仓库。
- 公寓管理、户型管理的新增、编辑、删除、状态切换改为写入运行时 mock 数据仓库。
- 首页、地图页、资料编辑页改为进入页面时动态读取统一数据，避免拿到旧快照。
- 公寓管理和户型管理新增批量导出、批量导入：
  - 批量导出：将当前列表 JSON 复制到剪贴板。
  - 批量导入：粘贴 JSON 后按 `id` 合并新增或更新。
- 查询层新增管理员运行时写入方法，仍不接真实接口、不写云数据库。

## 同步效果

- 管理员新增启用状态的公寓后，返回首页会在列表中显示。
- 管理员新增户型后，对应公寓详情可以读取到该户型。
- 地图页和资料页的公寓选项也改为动态读取运行时数据。

## 验证

- 已运行全量 JS 语法检查：`rg --files -g '*.js' miniprogram cloudfunctions | xargs -n1 node --check`。
- 已用 Node smoke test 验证：
  - 新增公寓后首页可读取。
  - 新增户型后公寓详情可读取。
  - 批量导入公寓后首页可读取。

## 约束

- 运行时 mock 数据只在当前小程序运行会话内有效，刷新/重新编译后恢复静态种子数据。
- 本轮未接 Supabase、云函数、本地缓存或真实文件导入导出。
