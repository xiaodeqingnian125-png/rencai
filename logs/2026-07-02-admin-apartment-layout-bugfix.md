# 2026-07-02 公寓管理布局乱码修复

## 问题

- 微信开发者工具中公寓管理页卡片标题被右侧操作按钮挤压，出现单字竖排。
- 顶部 `+ 新增` 和卡片 `编辑/删除` 使用 `button` 组件时宽度异常拉伸，导致页面看起来像“乱码”。

## 修复

- 将公寓管理主界面的 `+ 新增`、`编辑`、`删除` 从 `button` 改为可点击 `view`，避开小程序按钮默认样式干扰。
- 为顶部搜索区和新增入口设置固定宽度分配，搜索框使用 `width: 0` 配合 `flex: 1` 防止溢出。
- 为卡片标题与操作区设置明确宽度：标题保留剩余空间，操作区固定为两个 98rpx 胶囊按钮。
- 去掉公寓页关键区域对 `gap` 和 `inline-flex` 的依赖，改用显式 `margin`，提升基础库兼容性。

## 验证

- 已运行 `node --check miniprogram/pages/admin/index.js`。
- 已运行 `git diff --check`。

## 备注

- 本次只修复 `type=apartments` 的公寓管理主列表布局，不影响其它 6 个管理员子页面。
- 仍建议在微信开发者工具 iPhone 模拟器中重新编译预览，确认标题不再竖排、按钮不再横向溢出。
