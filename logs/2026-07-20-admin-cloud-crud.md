# 管理员公寓与户型云端 CRUD

日期：2026-07-20

## 本次改动

- 公寓、户型管理员页面改为异步读取云端数据和下一条业务 ID。
- 户型列表同步读取公寓数据，将 `apartment_code` 映射为管理员可读的公寓名称。
- 新增、编辑、删除和状态更新全部调用 `db` 云端接口，不再写入本地 Mock 运行时数据。
- 保存前通过管理员数据适配器生成云端规范字段，避免写入 `rent`、`rooms` 等展示字段。
- 增加加载、失败重试、防重复操作状态；云端失败时保留表单和原列表，只在真实成功后提示成功。
- 公寓和户型表单增加公寓编号字段，保证手工新增记录可以建立稳定关联。
- 云端保存会拒绝重复公寓编号；户型保存会校验所属公寓并回填权威 `apartment_id`，避免孤立数据。
- 活动、服务、物品、评论、用户等未完成管理模块保留入口，但只显示“功能预览 · 即将开放”，不读取或修改数据。
