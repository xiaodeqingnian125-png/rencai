# 2026-07-02 管理页原型差距修复

## 问题原因

- 管理员子页面上一轮实现偏向“大卡片 + 大按钮”的移动端表单风格，和设计稿中紧凑后台管理页的节奏不一致。
- 微信小程序 `button` 默认样式在 `flex` 布局中会带来宽度、行高和外边距影响，导致筛选按钮和卡片操作按钮被撑成大块。
- 管理卡片的信息密度偏低，操作按钮纵向堆叠，明显偏离原型中的紧凑横排操作。

## 本次修复

- 收紧管理员页面筛选条：筛选项改为横向小胶囊，不再撑满半屏。
- 收紧卡片操作区：编辑、启用/通过/完成、停用/隐藏/关闭、删除按钮改为紧凑横排。
- 修正按钮类的 `display`、`width`、`margin`、`line-height`，避免微信默认按钮样式干扰。
- 微调卡片 padding、字段间距、标签间距和字体，提升与后台原型的接近度。
