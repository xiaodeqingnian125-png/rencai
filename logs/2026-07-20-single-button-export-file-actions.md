# 2026-07-20 单按钮导出文件操作

## 目标

在保留管理员页唯一“批量导出”入口的前提下，让导出的 CSV 文件有明确的后续操作，而不增加下载中心或导出历史入口。

## 实现

1. 云函数生成 CSV 后，小程序下载云文件到临时路径。
2. 下载成功后显示一次性“导出文件已生成”弹窗，包含文件名和两个动作：
   - **打开并保存文件**：调用 `wx.openDocument`，开启系统菜单；真机可在右上角保存或转发。
   - **转发到微信**：调用 `wx.shareFileMessage`，可选择文件传输助手或其他微信联系人。
3. 关闭弹窗后清除临时文件路径，不保存导出记录、不新增入口。

## 环境说明

- 真机微信是文件保存和转发的验收环境。
- 微信开发者工具不能把小程序文件直接写入 Mac 的“下载”目录；当系统能力不可用时，页面会提示在真机微信中完成操作，而不会误报“已保存”。
- 云函数 `rencai` 必须部署包含 `createExportFile` action 的最新版本，文件才能生成。

## 验证

- 新增工具层测试：下载、打开、转发，以及转发能力缺失的真实失败结果。
- 新增页面测试：下载成功后显示操作弹窗，两个操作分别传入正确的临时路径与文件名。
- 全量 Node 测试、修改文件语法检查和 `git diff --check` 均在本任务完成前执行。
