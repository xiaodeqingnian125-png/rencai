# 单按钮导出文件操作设计

## 目标

管理员在不增加“下载中心”或导出历史入口的前提下，通过现有唯一的“批量导出”按钮，明确完成 CSV 文件的打开、保存或转发。

## 范围与约束

- 仅修改公寓和户型管理的现有导出流程；不新增页面、tab、列表入口或导出记录。
- 云函数继续生成 CSV 并返回 `fileID`；前端下载文件后才显示可操作界面。
- 真机微信是保存或转发文件的目标环境。
- 微信开发者工具不能将小程序文件直接保存到 Mac 下载目录，必须明确提示这一限制，不能报告“已保存”。
- 保持微信小程序原生实现，不引入第三方库。

## 交互

1. 管理员点击既有“批量导出”。
2. 云函数成功生成 CSV，前端下载得到临时文件路径。
3. 显示操作弹窗：
   - 主操作“打开并保存”：使用 `wx.openDocument({ fileType: 'csv', showMenu: true })` 打开系统文件预览；真机用户可在右上角菜单保存或转发。
   - 次操作“转发到微信”：使用 `wx.shareFileMessage` 直接调起文件转发，管理员可选择文件传输助手。
   - 取消：关闭弹窗，不误报保存成功。
4. 如果当前运行环境不支持对应能力：
   - `openDocument` 不可用或失败：提示“文件已生成，请在真机微信中打开并保存”。
   - `shareFileMessage` 不可用或失败：提示“当前环境不支持直接转发，请使用打开并保存”。
   - 下载失败：提示“导出文件下载失败，请重试”。

## 模块边界

- `miniprogram/utils/csv-share.js`：只处理“下载云文件”“打开文件”“转发文件”，通过回调或 Promise 返回明确状态，不处理页面 UI。
- `miniprogram/pages/admin/index.js`：只负责导出、展示操作弹窗、调用工具方法以及关闭弹窗。
- `miniprogram/pages/admin/index.wxml/.wxss`：在既有管理员页内渲染操作弹窗；不提供新的长期入口。

## 数据流与失败处理

`批量导出 → createExportFile → cloud.downloadFile → 导出操作弹窗 → openDocument / shareFileMessage`。

任一阶段失败都保留当前管理页和唯一导出入口；不创建本地假文件、不复制未知内容到剪贴板、不显示“保存成功”。

## 验收

- 生成成功后出现“打开并保存”和“转发到微信”两个操作，而非自动打开后没有指引。
- 点击“打开并保存”时调用 `wx.openDocument` 且 `showMenu: true`。
- 点击“转发到微信”时调用 `wx.shareFileMessage`。
- 开发者工具能力缺失时显示真机提示，不显示保存成功。
- 现有 CSV 云端生成、下载失败提示和公寓/户型导出均保持可用。
