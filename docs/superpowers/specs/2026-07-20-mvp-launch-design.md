# 晓得青年极速上线版设计规格

> 日期：2026-07-20
> 目标窗口：2026-07-22 至 2026-07-23 提交微信审核
> 状态：设计已通过对话确认，等待文档复核后进入实施计划

## 1. 目标

在不继续扩张社区业务范围的前提下，将当前微信小程序收敛为可提交审核的首发版本：游客可以稳定浏览真实公寓、户型和地图数据；管理员可以在云端维护公寓、户型、封面图和平面图，并完成 CSV 批量导入导出。

首发版本保留完整产品入口，但所有尚未完成的业务模块必须明确标记为功能预览，不能展示会误导用户的假成功行为。

## 2. 全局约束

- 使用微信小程序原生 WXML、WXSS 和 JavaScript。
- 不引入第三方前端库，不使用 `web-view`。
- 首页底部导航继续使用原生 `tabBar`。
- 云模式下真实业务数据统一经过 `miniprogram/data/db.js` 和 `rencai` 云函数。
- 云请求失败时显示错误状态，不自动回退到 Mock 数据。
- 管理员权限只以云端当前 OPENID 对应用户的 `users.role === "admin"` 为准。
- 用户可见的写操作必须真实持久化；无法持久化的功能统一转为“功能预览”。
- 图片静态资源和上传结果使用项目约定的 assets 或微信云存储路径。
- 每项实施任务完成后在 `logs/` 中记录 Markdown 日志。

## 3. 首发功能范围

### 3.1 真实开放的用户功能

- 首页公寓列表、搜索和筛选。
- 公寓详情。
- 户型列表和户型详情。
- 地图选房和公寓位置展示。
- 加入社群。
- 政策攻略。
- 用户登录、登录恢复和退出登录。
- “我的”页面中的管理员入口。

### 3.2 真实开放的管理员功能

- 公寓列表、搜索、筛选、新增、编辑、删除、启用和停用。
- 户型列表、搜索、筛选、新增、编辑、删除、启用和停用。
- 公寓与户型 CSV 模板下载、批量导入、预览、确认、历史记录和错误报告。
- 公寓与户型 CSV 批量导出。
- 公寓与户型封面图上传。
- 公寓平面图名称、图片和顺序管理。

### 3.3 保留展示但不正式开放的功能

- 服务和活动。
- 借个锤子。
- 找室友。
- 消息。
- 收藏、评论和点赞。
- 我的活动、帖子、借入、借出和订单。
- 活动、服务、物品、评论和用户管理。

这些页面和入口继续保留，但必须遵循：

- 页面显示“功能预览 · 即将开放”。
- 示例内容明确标注为示例内容。
- 报名、发布、申请、下单、删除消息、确认完成等写操作统一拦截。
- 统一提示“该功能正在准备中，暂未正式开放”。
- 不显示“提交成功”“报名成功”“订单已创建”等假成功文案。
- 不向云数据库写入未开放业务数据。
- 未开放的管理员入口可以显示，但点击后不进入 Mock CRUD 页面。

## 4. 总体架构与数据流

### 4.1 公开数据读取

```text
首页 / 公寓详情 / 户型详情 / 地图
  → miniprogram/data/db.js
  → wx.cloud.callFunction("rencai")
  → apartments / room_types
```

公开页面只读取状态为启用或兼容旧数据无状态字段的记录。列表和详情必须有加载、空数据、未找到、网络失败和重试状态。

### 4.2 管理员数据维护

```text
管理员公寓或户型页面
  → db.getAdminDataset / saveAdminItem / deleteAdminItem / updateAdminItemStatus
  → rencai 云函数 requireAdmin
  → apartments / room_types
  → 操作成功后重新读取云端列表
```

管理页常规 CRUD、CSV 导入导出和用户侧页面必须使用同一份云数据库数据。不得继续出现常规 CRUD 写 Mock、导入导出写云端的混合状态。

### 4.3 图片上传

```text
管理员选择图片
  → wx.chooseMedia
  → 大小与类型校验
  → wx.cloud.uploadFile
  → cloud:// fileID
  → 管理员保存表单
  → saveAdminItem
```

云上传成功后才能更新待保存表单。上传失败必须保留原图片，不能将本地临时路径写入云数据库。

## 5. CSV 批量导入设计

### 5.1 通用流程

```text
下载模板
  → 管理员填写 UTF-8 CSV
  → 上传文件
  → 服务端解析与校验
  → 预览有效和无效数据
  → 管理员二次确认
  → 分批 upsert
  → 显示结果和错误报告
```

### 5.2 文件限制

- 首发只支持 UTF-8 CSV，不直接解析 `.xls` 或 `.xlsx`。
- 单个 CSV 文件不超过 2MB。
- 公寓导入每次最多10条需要地址解析的数据。
- 户型导入每次最多500条。
- 超出限制时不执行写入，提示管理员分批导入。

### 5.3 公寓导入

- 管理员必须填写公寓编码、名称和详细地址。
- 管理员不需要填写经纬度。
- 缺少有效坐标时，预览阶段先调用腾讯地图解析地址，失败后调用高德地图。
- 坐标必须位于郑州有效范围内。
- 两个平台都无法解析时，该行进入错误列表，不允许写入数据库。
- 如果导出文件中已有有效经纬度，再次导入时可以直接复用。
- 以 `apartment_code` 作为唯一键；已存在时更新，未存在时新增。

### 5.4 户型导入

- 管理员必须填写所属公寓编码和户型名称。
- 所属公寓编码必须已存在。
- 以 `apartment_code + name` 作为组合唯一键。
- 已存在时更新，未存在时新增。

### 5.5 任务状态与幂等

- 任务必须经过预览后才能确认。
- `processing` 状态在有效时间内拒绝重复确认。
- `completed`、`partial_failed` 和 `failed` 状态不能再次确认。
- 需要重新导入时必须创建新任务。
- 部分记录失败时保留成功写入的数据，并记录逐行错误。
- 错误报告包含行号、原因和经过长度限制的原始行数据。

## 6. CSV 导出设计

- 公寓和户型分别导出。
- 公寓支持按区域和状态筛选导出。
- 导出使用 UTF-8 BOM，保证 Excel 和 WPS 打开中文正常。
- 所有文本继续进行 CSV 引号转义和公式注入防护。
- 公寓 CSV 包含“封面图路径”和“平面图”字段。
- 户型 CSV 包含“封面图路径”字段。
- “封面图路径”保存 `cloud://` fileID 或 HTTPS 地址，不在 CSV 内嵌图片。
- 公寓“平面图”字段保存 JSON 数组，完整保留每张图的名称和云路径。
- 文件分享不可用或失败时复制完整 CSV 到剪贴板。
- 文件分享和剪贴板都失败时明确提示失败，不能提示已保存或已生成。

## 7. 封面图上传设计

- 公寓和户型各支持一张封面图。
- 选择图片时使用压缩图片，单张不超过5MB。
- 公寓封面上传到 `covers/apartments/`。
- 户型封面上传到 `covers/rooms/`。
- 上传成功后将云存储 fileID 写入表单；管理员点击保存后才写入数据库。
- 上传失败时保持原值并提示重试。
- 保存成功后，首页、详情和管理后台重新读取时显示同一图片。
- 首发不自动删除被替换的旧云文件，旧文件清理作为上线后的存储治理任务。

## 8. 公寓平面图设计

### 8.1 数据结构

`apartments` 文档增加 `floor_plans` 数组，顺序即展示顺序：

```js
floor_plans: [
  {
    name: "项目总平面图",
    image: "cloud://.../project-plan.jpg"
  },
  {
    name: "3号楼标准层平面图",
    image: "cloud://.../building-3.jpg"
  }
]
```

每一项必须同时具有非空名称和有效的 `cloud://` 或 HTTPS 图片路径。无有效数据时不显示平面图区。

### 8.2 公寓详情页

- 平面图区位于“周边服务”之后、“户型选择”之前。
- 标题为“平面图”。
- 详情页只展示数组中的前3张图片。
- 图片按照保存顺序纵向排列。
- 每张图片保持完整比例，不裁切主要内容。
- 图片名称位于图片正下方并水平居中。
- 点击图片使用 `wx.previewImage` 打开原生图片预览和缩放。
- 当有效图片数量大于3张时，标题右上角显示“更多”。
- 有效图片数量不超过3张时不显示“更多”。

### 8.3 更多平面图页

- 新增原生页面 `pages/apartment-plans/index`。
- 点击公寓详情页“更多”进入该页面，并传递公寓 ID。
- 页面重新通过公开公寓详情接口读取数据，不通过 URL 传输完整图片数组。
- 页面只展示第4张及之后的平面图。
- 图片逐张纵向排列，名称位于图片正下方并居中。
- 点击任意图片继续使用 `wx.previewImage`。
- 页面必须支持加载、无更多图片、未找到、网络失败和重试状态。

### 8.4 管理员维护

- 平面图属于公寓，不属于单个户型。
- 公寓编辑表单支持添加、删除、重命名和调整顺序。
- 每张图片上传到 `floor-plans/apartments/{apartment_code}/`。
- 上传成功后才更新对应项的 `image`。
- 公寓保存时一次性提交规范化后的 `floor_plans` 数组。
- 管理员保存顺序决定详情页前3张和“更多”页的排列顺序。

## 9. 错误处理

- 所有页面异步回调在卸载后不得继续 `setData`。
- 请求竞争时旧响应不得覆盖新响应。
- 管理员写操作期间禁用重复点击。
- 云函数失败时保留当前表单和图片，不清空管理员输入。
- 权限失效统一提示“仅管理员可执行”，并阻止写入。
- 地址解析失败必须定位到具体 CSV 行，不能保存为 `(0, 0)` 后伪装成功。
- 图片路径不合法时公开接口返回空字符串，页面使用现有渐变占位图。
- 平面图数组中的无效项在返回页面前过滤。

## 10. 验收标准

### 10.1 用户端

- 游客可从首页进入公寓详情和户型详情。
- 首页、公寓详情、户型详情和地图均读取云端真实数据。
- 公寓详情正确展示0张、1张、3张和超过3张平面图的状态。
- 图片名称位于图片正下方并居中。
- 超过3张时“更多”入口可进入独立页面，并只展示剩余图片。
- 断网、无数据和非法 ID 不白屏。
- 未开放模块都有功能预览标识，所有写操作均不产生假成功。

### 10.2 管理员端

- 普通用户无法读取或修改管理员数据集。
- 管理员公寓和户型 CRUD 全部写入云数据库。
- 管理员修改后重新进入用户页面可以看到最新数据。
- 公寓和户型封面图上传失败时不会保存本地临时路径。
- 平面图可以添加、命名、排序、删除和保存。
- CSV 导入经过预览和二次确认，重复确认被阻止。
- 公寓只填地址即可获得有效坐标；失败行不会写入。
- 导出 CSV 包含正确的封面图路径和平面图 JSON。
- 导出 CSV 再导入后，图片路径、平面图名称和顺序保持一致。

### 10.3 发布门槛

- 所有 JavaScript 语法检查通过。
- 所有 JSON 文件解析通过。
- `app.json` 页面文件完整。
- `git diff --check` 通过。
- 微信开发者工具编译无错误。
- iPhone SE、iPhone 15 模拟器无 P0/P1 显示问题。
- 至少一台 iPhone 和一台 Android 真机完成登录、地图、图片上传和 CSV 分享验证。
- 普通用户与管理员权限回归通过。
- P0 和 P1 缺陷为零后才能上传审核版本。

## 11. 实施顺序

1. 固化并验证当前地图和 CSV 未提交改动。
2. 统一管理员公寓和户型 CRUD 到云端数据层。
3. 修复云图片上传失败时写入临时路径的问题。
4. 完成地址自动解析和公寓导入上限调整。
5. 完成封面路径和平面图 JSON 的 CSV 往返。
6. 实现平面图数据、详情展示、更多页面和管理员维护。
7. 给未开放模块增加统一预览态并拦截假写操作。
8. 完成静态检查、开发者工具回归、真机验收和云函数部署。

## 12. 不在首发范围

- 微信支付。
- 微信订阅消息。
- 真实活动发布、审核与报名。
- 真实借物发布、申请与归还。
- 真实找室友发布和审核。
- 真实站内消息读写。
- 收藏、评论和点赞云端闭环。
- 旧云存储文件自动清理。
- Web 管理后台。
