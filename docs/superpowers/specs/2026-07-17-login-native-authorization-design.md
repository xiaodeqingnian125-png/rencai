# 登录界面昵称和手机号原生授权设计

## 背景

当前登录界面中，昵称为纯手动输入，手机号通过手动输入 + 右侧「一键获取」按钮双模式获取。用户希望移除「一键获取」按钮，改为点击输入框本身即触发微信授权获取。

## 目标

- 昵称输入框点击后可链接到微信提供的昵称
- 手机号输入框点击后可触发微信授权获取手机号
- 移除独立的「一键获取」按钮
- 保留手动输入作为兜底方案

## 设计

### 昵称输入

将 `<input type="text">` 改为 `<input type="nickname">`。微信原生组件，点击后键盘上方自动显示用户微信昵称，点击可快速填充，也可手动输入。

### 手机号输入

移除独立的「一键获取」按钮，改为按钮伪装成输入框的方式：

- **微信获取模式（默认）**：`<button open-type="getPhoneNumber">` 样式与输入框一致，placeholder「点击获取微信手机号」
- **手动输入模式**：真实 `<input type="number">`，授权失败或用户主动选择时切换
- 底部小文字链接切换模式：微信模式显示「无法获取？手动输入」，手动模式显示「使用微信获取」

### 组件状态

`login-modal` 的 `data` 新增 `phoneInputMode` 字段（`'wechat'` 或 `'manual'`），默认 `'wechat'`。

### 错误处理

- 昵称为空禁止登录
- 手机号授权失败 → 自动切换手动模式并提示
- 手机号不足 11 位禁止登录

## 影响范围

仅修改 `components/login-modal/` 组件（index.wxml、index.js、index.wxss），不影响其他页面。
