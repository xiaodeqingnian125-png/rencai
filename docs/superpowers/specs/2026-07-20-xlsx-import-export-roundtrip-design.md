# Excel 文件导入导出闭环设计

## 目标

管理员完成“批量导出 `.xlsx` 文件 → 在 Excel/WPS 修改 → 批量导入选择同一 `.xlsx` 文件 → 预览并确认导入”的真实文件闭环。

## 用户流程

1. 管理员在公寓或户型管理页点击既有“批量导出”。
2. 云函数生成真正的 `公寓导出.xlsx` 或 `户型导出.xlsx`，字段顺序沿用当前 CSV 表头。
3. 小程序下载文件后调用微信原生文档页；管理员通过右上角系统菜单保存到手机“文件”。不显示“转发到微信”按钮。
4. 管理员用 Excel/WPS 修改此文件并保留 `.xlsx` 格式。
5. 点击既有“批量导入”，选择 `.xlsx` 文件；小程序上传原始文件到云存储，云函数解析第一张工作表，转换为当前 CSV 校验管线使用的表格文本。
6. 复用现有预览、地理编码校验、确认导入和历史记录能力。

## 兼容性与边界

- `.xlsx` 是新的标准导入导出格式；原有 `.csv` / `.txt` 导入保持兼容。
- 不伪造 `.xls` 扩展名，也不把 CSV 内容包装成假 Excel 文件。
- 真机微信仍需管理员在系统文档页点“保存”；小程序不能静默写入手机或 Mac 的公共文件目录。
- 开发者工具不能验证保存到 Mac 下载目录；真机是文件保存验收环境。
- 仅处理第一个工作表，最多 500 条数据行，保留当前 CSV 的字段校验、公式注入防护和地址解析限制。
- 文件名、表头和数据格式对公寓与户型保持一一对应，导出文件可不改列名直接回传导入。

## 架构

- 云函数增加 `xlsx` 依赖：导出时由 CSV 文本生成真正的 workbook；导入时从云存储读取上传的 `.xlsx` buffer，解析首个 worksheet 并转换为 CSV 文本。
- 新管理员云函数 action `createImportTaskFromFile` 仅接收可信的云存储 `fileID` 与类型，不接受客户端转换后的 Excel 文本；解析完成后复用 `createImportTask`。
- 小程序导出工具以 `xlsx` 文件类型调用 `wx.openDocument`；移除导出完成后的自定义“打开/转发”选择弹窗。
- 小程序导入页识别 `.xlsx` 并上传文件到 `imports/`；`.csv` / `.txt` 继续原样读取后调用原 action。

## 失败处理

- Excel 文件无工作表、无有效表头、超过 500 行或解析失败：显示明确错误，不创建可确认的导入任务。
- 文件上传失败：停留在管理页并提示上传失败。
- 文档打开失败：提示“文件已生成，请在真机微信中打开并保存”，不显示保存成功。
- 云函数生成/解析失败：返回统一错误给管理员，不伪造文件或导入成功。

## 验收

- 云存储导出文件扩展名和内容均为真实 `.xlsx`。
- 微信打开文档调用使用 `fileType: "xlsx"` 和 `showMenu: true`。
- 导出弹窗中不再显示“转发到微信”。
- 选择修改后的 `.xlsx` 能进入现有预览页面，字段名和数据与导出文件一致。
- 旧 CSV 文件仍可直接选择导入。
