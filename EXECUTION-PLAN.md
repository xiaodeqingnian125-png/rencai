## 验收标准

本标准为发布前硬性门槛。每项标准标注"可测量指标"，验收时逐项检查，不达标记录为缺陷。

### 一、视觉还原度

所有页面与 `rencai-ui/` 设计稿对照，误差不超过以下阈值。

#### 1.1 色彩系统

小程序实际使用的色值必须与设计稿 CSS 变量完全一致，不允许近似值。

| 设计稿变量 | 色值 | 用途 |
|-----------|------|------|
| `--bg` | `#fafaf6` | 页面背景 |
| `--surface` | `#fffdf7` | 卡片背景 |
| `--surface-warm` | `#fef8ef` | 暖色卡面 |
| `--fg` | `#2d2318` | 主文字 |
| `--fg-2` | `#5c4a38` | 次级文字 |
| `--muted` | `#8c7a68` | 辅助文字 |
| `--meta` | `#a69682` | 元信息文字 |
| `--border` | `#e8e0d4` | 边框 |
| `--border-soft` | `#f0eae0` | 轻边框 |
| `--accent` | `#e8723c` | 主色（陶土橙） |
| `--accent-hover` | `#f0804a` | 主色悬停 |
| `--accent-active` | `#d4622c` | 主色按下 |
| `--price-orange` | `#e8590c` | 价格标签 |
| `--community-green` | `#5b9a6f` | 社区/活动 |
| `--map-blue` | `#5b8cb8` | 地图/地铁 |
| `--badge-red` | `#e04a3a` | 红点/未读 |

可测量指标：用微信开发者工具的 Wxml 面板检查实际渲染色值，与上表逐项比对，色值差异 = 0 方为通过。

#### 1.2 间距体系

设计稿采用 8px 基准网格，小程序用 rpx 表达。换算关系：750rpx = 375px（iPhone 6 基准），因此 1px = 2rpx。

| 设计稿间距 | rpx 值 | 允许误差 |
|-----------|--------|---------|
| 8px | 16rpx | ±0 |
| 12px | 24rpx | ±0 |
| 16px | 32rpx | ±0 |
| 20px | 40rpx | ±0 |
| 24px | 48rpx | ±0 |

可测量指标：页面内任意相邻元素的间距必须落在上述网格点上，不允许出现 18rpx、30rpx 等非网格值。卡片内边距统一为 32rpx，卡片间距统一为 24rpx。

#### 1.3 圆角体系

| 设计稿变量 | 像素值 | rpx 值 | 用途 |
|-----------|--------|--------|------|
| `--radius-sm` | 8px | 16rpx | 小标签/小按钮 |
| `--radius-md` | 12px | 24rpx | 输入框/中型卡片 |
| `--radius-lg` | 18px | 36rpx | 大卡片/底部弹窗 |
| `--radius-pill` | 980px | 999rpx | 胶囊按钮/圆头像 |

可测量指标：所有圆角值必须与上表完全匹配。不允许出现 20rpx、28rpx 等非体系值。

#### 1.4 字体层级

| 层级 | 设计稿字号 | rpx 值 | 字重 | 用途 |
|------|-----------|--------|------|------|
| 标题大 | 21px | 42rpx | 700 | 页面主标题 |
| 标题 | 16px | 32rpx | 650-700 | 卡片标题/区块标题 |
| 正文 | 14px | 28rpx | 400 | 主体内容 |
| 辅助 | 12px | 24rpx | 400-500 | 标签/元信息 |

可测量指标：字体族统一为 `-apple-system, "PingFang SC"`。任何页面不得出现小于 24rpx 的文字。标题与正文的字重差至少为 2 级（如 700 vs 400）。

#### 1.5 阴影与层次

设计稿定义：`--elev-raised: 0 12px 32px rgba(0,0,0,0.06)`

rpx 换算：`0 24rpx 64rpx rgba(0,0,0,0.06)`

可测量指标：
- 卡片阴影仅出现在需要抬升层次的场景（浮层、弹窗、底部栏），普通卡片用边框 `1rpx solid #f0eae0` 而非阴影
- 阴影透明度不超过 0.08，不允许出现硬边阴影
- 底部固定栏（如报名按钮栏）阴影方向向上：`0 -8rpx 24rpx rgba(0,0,0,0.04)`

#### 1.6 图片与占位

当前项目无真实图片资源，使用渐变色块占位。设计稿定义了 6 组暖色渐变作为公寓卡片封面。

可测量指标：
- 每张公寓卡片封面使用设计稿定义的渐变色组之一，不允许出现纯色块或未定义的渐变
- 渐变色组之间在同一页面内混排，避免连续两张卡片使用相同渐变
- 封面区域内叠加半透明文字层（标题、价格），文字对比度 ≥ 4.5:1（WCAG AA）
- 图标统一使用 SVG 格式，线条粗细 1.5px，颜色随语义变化（主色/次色/辅助色）

### 二、交互与动画

#### 2.1 缓动曲线与时长

设计稿定义：`--ease-standard: cubic-bezier(0.28, 0, 0.22, 1)`

微信小程序对应：`cubic-bezier(0.28, 0, 0.22, 1)`

| 动效类型 | 设计稿时长 | 实际值 | 用途 |
|---------|-----------|--------|------|
| 快速反馈 | 150ms | 150ms | 按钮按下/标签切换/图标状态 |
| 标准过渡 | 220ms | 220ms | 弹窗展开/卡片展开/页面切换 |

可测量指标：
- 所有 `transition` 必须使用 `cubic-bezier(0.28, 0, 0.22, 1)`，不允许 `linear` 或 `ease`
- 按钮点击反馈延迟 ≤ 100ms（从手指触碰到视觉变化）
- 弹窗从触发到完全展开 ≤ 220ms，展开动画为从底部滑入 + 淡入
- 底部弹窗（sheet）必须有半透明遮罩，遮罩淡入时长 150ms

#### 2.2 触控反馈

可测量指标：
- 所有可点击元素（按钮、卡片、列表项）必须有 `hover-class` 或 `active` 状态
- 按下时视觉变化：背景色加深一档（如 `#fffdf7` → `#fef8ef`）或透明度降至 0.85
- 按下反馈面积不缩小（不允许按下时元素变小）
- 列表项按下时左侧出现 6rpx 主色条或整体背景变色

#### 2.3 页面切换

可测量指标：
- `navigateTo` 页面从右侧滑入，时长 300ms（微信默认）
- `switchTab` 页面淡入切换，无闪烁
- 返回上一页时数据刷新无延迟（`onShow` 中数据更新后 `setData` 在 100ms 内完成）
- 页面首次加载时显示骨架屏或 loading 态，数据就绪后平滑替换，不允许出现空白闪烁

#### 2.4 表单交互

可测量指标：
- 输入框获焦时边框变为 `--accent` 色，时长 150ms
- 键盘弹出时页面自动上推，输入框不被遮挡
- 提交按钮在表单未填完时为禁用态（降低透明度至 0.4，不可点击）
- 表单提交后按钮显示 loading 态，防止重复提交
- 手机号输入限制 11 位数字，超出截断

### 三、数据一致性

#### 3.1 运行时写入同步

可测量指标：
- 任意写操作（报名/评论/收藏/借用/发帖/下单）完成后，在 0ms 延迟内通过 `onShow` 或回调刷新关联页面数据
- 管理员新增公寓后，首页列表、地图页、筛选结果在下次 `onShow` 时读取到新数据
- 用户报名活动后，活动详情页人数 +1、报名按钮变"已报名"、消息页出现报名通知、个人中心"我的活动"出现记录——四处同步，无遗漏
- 收藏切换后，详情页收藏态、首页卡片收藏角标、个人中心"我的收藏"列表三处同步

#### 3.2 边界数据

可测量指标：
- 空列表状态：显示设计稿定义的空状态组件（图标 + 文案），不允许空白
- 搜索无结果：显示"未找到相关公寓"文案 + 推荐内容
- 活动满员：报名按钮变"已满员"禁用态，不可点击
- 重复操作：重复报名提示"你已报名该活动"，重复收藏执行取消收藏
- 长文本截断：公寓名超过 1 行省略号截断，评论正文超过 3 行省略号截断

### 四、性能

#### 4.1 渲染性能

可测量指标：
- 首页首次渲染完成（FCP）≤ 1s（微信开发者工具 Performance 面板）
- 列表滚动帧率 ≥ 55fps（长列表使用 `scroll-view` 虚拟渲染或限制 DOM 节点数）
- `setData` 单次数据量 ≤ 256KB，避免大对象全量传输
- 图片占位渐变使用 CSS 实现，不使用图片资源加载

#### 4.2 交互响应

可测量指标：
- 任意按钮点击到反馈 ≤ 100ms
- 页面跳转触发到新页面首次渲染 ≤ 500ms
- 弹窗展开/关闭 ≤ 220ms
- 输入框打字延迟 ≤ 50ms（不卡顿）

- 管理员身份在重新打开小程序后保持（本地缓存恢复登录态）

#### 6.3 登录态持久化

可测量指标：
- 登录后关闭小程序再打开，仍为登录状态
- 退出登录后所有个人数据清空，管理员面板隐藏
- 登录态过期或缓存清除后，降级为游客模式，不报错

### 七、细节打磨

以下为"好"与"优秀"的分水岭，逐项检查。

#### 7.1 文案规范

可测量指标：
- 所有 Toast 文案 ≤ 14 字，完整可见
- 所有 Modal 弹窗标题 ≤ 8 字，正文 ≤ 60 字
- 空状态文案有人情味（如"还没有活动，先去看看公寓吧"），非"暂无数据"
- 错误提示具体可行动（如"手机号需 11 位数字"，非"输入有误"）

#### 7.2 图标一致性

可测量指标：
- 全站图标线条粗细统一为 1.5px（或 24px 画布上 1.5px stroke）
- 图标圆角风格统一（全站同为圆角端点 `round` 或同为直角端点）
- 功能相同图标在不同页面保持一致（如收藏图标在详情页和列表页外观相同）
- 选中态图标为填充色，未选中态为描边色

#### 7.3 列表与卡片节奏

可测量指标：
- 列表项高度在同一列表内一致（不因内容长短导致高度跳动）
- 卡片间距在页面内统一（同一页面所有卡片间距相同）
- 长列表底部留白 ≥ 48rpx，避免最后一项贴底
- 列表数据加载中显示骨架屏，骨架屏形状与最终内容形状一致

#### 7.4 弹窗与遮罩

可测量指标：
- 底部弹窗圆角顶部 36rpx，底部直角
- 弹窗顶部有把手（handle）指示可滑动关闭
- 遮罩层不可穿透点击（`catchtap` 阻止冒泡）
- 弹窗内容超出屏幕时可滑动，不截断

#### 7.5 按钮与可操作区域

可测量指标：
- 所有可点击元素最小触控区域 ≥ 88rpx × 88rpx（44pt × 44pt，Apple HIG）
- 主操作按钮高度 ≥ 88rpx，宽度铺满或居中
- 按钮间距 ≥ 24rpx，避免误触
- 禁用态按钮透明度 0.4，不可点击，视觉明确区分

### 八、管理员后台

#### 8.1 CRUD 完整性

可测量指标：
- 7 项管理（公寓/户型/活动/服务/物品/评论/用户）均支持新增、编辑、删除
- 新增/编辑表单字段与设计稿一致，必填项有标识
- 删除操作有二次确认弹窗，防止误删
- 列表支持搜索和筛选（与设计稿一致）
- 数据变更后列表实时刷新，无需手动 reload

#### 8.2 导入导出

可测量指标：
- 支持 CSV 批量导入（公寓/户型/活动/服务/物品）
- 导入时显示进度和结果（成功 N 条/更新 N 条/忽略 N 条）
- 导出为 CSV 格式，文件名包含日期

## 验收流程

### 逐页验收清单

对每个页面执行以下检查：

1. 在 iPhone 15 模拟器下打开页面，与设计稿截图并排对比
2. 检查配色/间距/圆角/字号是否符合"视觉还原度"标准
3. 逐个触发所有交互（点击/滑动/输入/提交），检查反馈是否符合"交互与动画"标准
4. 制造边界情况（空数据/满员/重复操作/长文本），检查处理是否符合"数据一致性"标准
5. 在 iPhone SE 模拟器下重复 1-4，检查适配性
6. 记录所有偏差为缺陷，按严重程度分级

### 缺陷分级

| 级别 | 定义 | 处置 |
|------|------|------|
| P0 | 功能不可用/崩溃/白屏 | 必须修复后方可发布 |
| P1 | 视觉明显偏差/交互卡顿/数据不同步 | 必须修复后方可发布 |
| P2 | 细节瑕疵/文案不佳/边缘 case | 发布后限时修复 |
| P3 | 优化建议 | 纳入下一迭代 |

### 发布门槛

- P0 缺陷数 = 0
- P1 缺陷数 = 0
- P2 缺陷数 ≤ 5（且无功能性影响）
- 所有页面通过逐页验收清单
- iPhone SE / 13 mini / 15 / 15 Pro Max 四档机型均无 P0/P1 缺陷
